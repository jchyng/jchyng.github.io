# 경기도 소방재난본부 홈페이지 & 백오피스 리뉴얼

경기도 소방재난본부의 **대국민 홈페이지**([119.gg.go.kr](https://119.gg.go.kr/main))와 **관리자용 백오피스(CMS)**를 리뉴얼하는 7개월간의 실무 프로젝트입니다. 신기술 도입 제안(Meilisearch)부터 폐쇄망 온프레미스 인프라 구축·안정화까지 전 영역에 걸쳐 핵심 역할을 담당했습니다.

![경기도 소방재난본부 홈페이지 메인](/images/firestation-main.png)

---

## 🏆 핵심 성과

- **검색 성능 99% 개선**: Meilisearch 도입으로 10만 건 이상 공공 콘텐츠 통합 검색 속도를 평균 **50ms 이하**로 단축, 기술 가이드 사내 자산화
- **폐쇄망 인프라 구축**: 경기도 소방재난본부 서버실 환경에서 Apache-Tomcat **이중화 구성 및 세션 클러스터링** 직접 구축
- **네트워크 레이어 장애 해결**: tcpdump 패킷 분석으로 에러 로그 없는 VM 세션 클러스터링 소스 IP 미인식 문제의 근본 원인 추적·해결
- **공공 표준 준수 CMS 개발**: 전자정부프레임워크(Spring eGov) 기반 권한 관리, 콘텐츠 이력 관리, 법제처 API 연동 기능 구현

## 🔍 Deep Dive 1. Meilisearch 도입 — 검색 성능 99% 개선과 기술 자산화

기존 RDBMS 기반의 `LIKE` 검색은 데이터가 누적될수록 성능이 급격히 저하되는 구조였습니다. 공공기관 특성상 별도 검색 서버를 추가하기 어려운 **단일 서버 환경**이라는 제약 속에서, 최적의 솔루션을 직접 조사하여 **Meilisearch**를 제안하고 도입했습니다.

![Meilisearch 통합 검색 결과 화면 — 키워드 하이라이팅, 메뉴/게시글/콘텐츠 카테고리 분류](/images/firestation-search.png)

**성과**

| 항목           | 내용                                                                            |
| -------------- | ------------------------------------------------------------------------------- |
| 검색 응답 속도 | 10만 건 이상 데이터에서 평균 **50ms 이하** (기존 대비 약 **99% 개선**)          |
| 검색 품질      | 오타 허용(Typo-tolerance), 키워드 하이라이팅, 메뉴·게시글·콘텐츠 카테고리 분류 |
| 조직 기여      | **도입 매뉴얼 작성 후 사내 기술 블로그 공유** → 타 사업팀에서도 동일 기술 채택  |

> 기술 도입 과정을 문서화하여 팀의 학습 비용을 줄이고, **개인의 경험을 조직의 자산으로 전환**했습니다.

## 🖥️ Deep Dive 2. 폐쇄망 온프레미스 인프라 구축

경기도 소방재난본부 서버실(폐쇄망) 환경에서 **Apache-Tomcat 이중화 구성**과 **All-to-All 세션 클러스터링**을 직접 구축했습니다. 외부 인터넷 접속이 차단된 환경이므로 의존성 패키지부터 설정 파일까지 모든 것을 수작업으로 준비하고 검증해야 했습니다.

```
[Apache httpd] ─── mod_jk (AJP) ──┬── [Tomcat WAS 1]
                                   └── [Tomcat WAS 2]

세션 클러스터링: All-to-All (DeltaManager)
포트: AJP 8009, 세션 복제 4000
```

## 🔧 Deep Dive 3. tcpdump로 에러 로그 없는 장애를 해결하다

세션 클러스터링 구성 직후, **VM 환경에서만 세션이 공유되지 않는 문제**가 발생했습니다. 에러 로그가 전혀 출력되지 않아 원인 파악이 극히 어려운 상황이었습니다.

**문제 추적 과정**

로그가 없는 상황에서 선택할 수 있는 방법은 하나뿐이었습니다. **tcpdump로 서버 간 패킷 흐름을 직접 분석**했습니다.

```bash
# VM 환경에서 서버 간 세션 복제 패킷 캡처
tcpdump -i eth0 -n host [상대방 WAS IP] and port 4000
```

**원인 발견**

VM 환경에서 자기 자신의 IP를 조회하면 실제 사설 IP 외에 `127.0.0.1`과 `10.0.2.1`이 함께 감지됩니다. 클러스터링 라이브러리가 이 중 잘못된 IP를 송신 주소로 선택하면서, **세션 복제 메시지가 외부로 나가지 않고 자기 자신에게 루프**되는 현상이 발생한 것이었습니다.

**해결**

```xml
<!-- server.xml 수정 전: auto로 IP를 자동 감지 -->
<Receiver className="..." address="auto" port="4000" />

<!-- server.xml 수정 후: VM 내부 IP로 명시 고정 -->
<Receiver className="..." address="[VM 사설 IP]" port="4000" />
```

`Receiver` 주소를 `auto`에서 **VM 사설 IP로 명시적으로 고정**하여 문제를 완전히 해결했습니다.

> 에러 로그조차 없는 장애를 네트워크 레이어에서 직접 추적하여 해결한 이 경험은, **인프라와 네트워크 지식이 개발자의 문제 해결 범위를 얼마나 넓혀주는지**를 실감하게 해준 계기였습니다.

## 📋 CMS 백오피스 핵심 기능 구현

관리자의 실무 운영에 직결되는 기능들을 전자정부프레임워크(Spring eGov) 기반으로 설계부터 구현까지 담당했습니다.

- **콘텐츠 이력 관리**: 모든 수정 이력을 기록하여 감사(Audit) 추적 가능
- **메뉴 인터셉터 기반 권한 제어**: URL 패턴별 접근 권한을 DB에서 관리하여 유연한 권한 설계
- **법제처 API 연동**: 소방 관련 법령 태그 선택 시 해당 법령의 개정 이력을 실시간 조회하는 소방관련법규 페이지 구현
- **NICE 본인인증 연동**: SDK 기반 휴대폰 본인인증 흐름 구현

![소방관련법규 페이지 — 법령 태그 클릭 시 법제처 API로 개정 이력 실시간 조회](/images/firestation-law.png)

법령 버튼(위험물안전관리법, 소방기본법 등 40여 개)을 클릭하면 법제처 Open API를 호출해 해당 법령의 법령명, 소관부처, 제개정 구분, 공포번호, 시행일 등을 테이블로 렌더링합니다. 페이지 로드 시 불필요한 전체 조회 없이 **선택한 법령에 대해서만 API를 호출**하는 구조로 구현했습니다.

```java
// 메뉴별 권한 체크 인터셉터 (개념 코드)
@Component
public class MenuAuthInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, ...) {
        String requestUri = request.getRequestURI();
        String userRole   = (String) request.getSession().getAttribute("userRole");

        // DB에서 해당 URI에 허용된 권한 목록 조회 후 비교
        boolean hasPermission = menuService.hasPermission(requestUri, userRole);

        if (!hasPermission) {
            response.sendRedirect("/error/403");
            return false;
        }
        return true;
    }
}
```

## 📝 회고

- **제안하는 개발자**: 주어진 요구사항만 구현하는 것이 아니라, 성능 문제를 먼저 발견하고 Meilisearch라는 해결책을 직접 조사·제안한 경험은 **개발자가 비즈니스 가치에 능동적으로 기여할 수 있다**는 것을 보여준 사례였습니다.
- **레이어를 넘나드는 문제 해결**: 에러 로그 없이 tcpdump로 근본 원인을 찾아낸 경험은, 애플리케이션 코드를 넘어 **네트워크·인프라 레이어까지 사고하는 폭**을 키워주었습니다.
- **경험의 조직 환원**: Meilisearch 매뉴얼을 사내에 공유하여 타 팀에서도 같은 기술을 채택하게 된 것처럼, **개인의 학습을 팀 자산으로 전환하는 것**이 개발자의 중요한 역할 중 하나임을 깨달았습니다.
